An important reason for using LaTeX is that it is fairly easy to typeset mathematical expressions. For basic mathematics, no packages are needed. We will use @mathtools@, an extension of @amsmath@, since we will typeset more complex expressions:

{% highlight latex %}
\usepackage{mathtools}
{% endhighlight %}

This lesson we will discuss some of the most used commands for typesetting mathematics. For more details you can look at this "Short Math Guide for LaTeX":ftp://ftp.ams.org/ams/doc/amsmath/short-math-guide.pdf and the documentation of "amsmath":http://www.ctan.org/pkg/amsmath or "mathtools":http://www.ctan.org/pkg/mathtools.

h2. Mathematical environments

Mathematics within text like $1+1=2$ is called *inline* mathematics. A line that consists of a single mathematical expression like
\[
1+1=2
\]
is called a *display*. Inline mathematics can be typeset between @$@s.

<div class="example">
We know that $1+1=2$.
</div>

{% highlight latex %}
We know that $1+1=2$.
{% endhighlight %}

Mathematics can be typeset in a display by putting it between @\[@ and @\]@.


<div class="example">
Examine the curve given by
\[
y = x^2 + 2.
\]
Does it attain a minimum?
</div>

{% highlight latex %}
Examine the curve given by
\[
y = x^2 + 2.
\]
Does it attain a minimum?
{% endhighlight %}

Recall that an empty line creates a new paragraph. Therefore there are no empty lines before or after the display in this example.

To *number* displays automatically one uses the @equation@ environment. We will see later how to reference displays.

<div class="example">
Since both $a$ and $b$ are odd we have
\begin{equation}
a + b = 2x
\end{equation}
and
\begin{equation}
a-b = 2y
\end{equation}
for some $x$ and $y$.
</div>

{% highlight latex %}
Since both $a$ and $b$ are odd we have
\begin{equation}
a + b = 2x
\end{equation}
and
\begin{equation}
a-b = 2y
\end{equation}
for some $x$ and $y$.
{% endhighlight %}

 LaTeX switches to *math mode* in every mathematical environment. The most important differences compared to _text mode_ are:
* Spaces and line breaks will be ignored; whitespace will be determined by the operators used or by special commands.
* Blank lines are not permitted.
* A single letter will be considered the name of a variable and will be typeset accordingly. If you wish to use normal text in _math mode_ you have to use a special command.

In the rest of this lesson we will assume that you are working in _math mode_.

h2. Powers and indices

Powers and indices can be inserted using the hat @^@ and the underscore @_@. For example

<div class="example">
\[
a^2\quad a_i\quad a_i^2
\]
</div>

{% highlight latex %}
a^2  a_i  a_i^2
{% endhighlight %}

The @_@ and @^@ bind themselves to the next _group_. Recall that groups can be made with @{@ and @}@. Hence

<div class="example">
\[
a^{n+1}\quad a^n+1\quad a_{i/2}
\]
</div>

{% highlight latex %}
a^{n+1}  a^n+1  a_{i/2}
{% endhighlight %}

h2. Operators

An *operator* is a function written as a word, like $\sin$ and $\log$. LaTeX has many predefined operators, but sometimes it is necessary to define one yourself. We create operators using @\DeclareMathOperator@ in the preamble.

<div class="example">
\[
\DeclareMathOperator{\tr}{tr}
\DeclareMathOperator{\ord}{ord}
\DeclareMathOperator{\Poisson}{Poisson}
\tr A,\quad \ord_p{a} = 2,\quad \Poisson(\lambda)
\]
</div>

{% highlight latex %}
\documentclass[a4paper]{article}
\usepackage{mathtools}
\DeclareMathOperator{\tr}{tr}
\DeclareMathOperator{\ord}{ord}
\DeclareMathOperator{\Poisson}{Poisson}

\begin{document}
\[
\tr A, \ord_p a = 2, \Poisson(\lambda)
\]
\end{document}
{% endhighlight %}


h2. Common expressions

A *fraction* can be created using @\frac{2}{3}@. A *binomial coefficient* is made with @\binom{n}{k}@. It is possible to create fractions inside fractions.

<div class="example">
\[
\frac{\frac{1}{x}+\frac{1}{y}}{y-z} 
\]
</div>

{% highlight latex %}
\frac{\frac{1}{x}+\frac{1}{y}}{y-z}
{% endhighlight %}

We can use @\tfrac@ and @\tbinom@ for small fractions and binomial coefficients. It is up to the user to decide when to use @\tfrac@.

*Roots* can be inserted as @\sqrt{2}@ or @\sqrt[n]{2}@ for $\sqrt[n]{2}$.

A *sum* or *integral* is given by @\sum@ or @\int@, use @^@ and @_@ to specify the boundaries. For example

<div class="example">
\[
\sum_{i=1}^{10} t_i\quad \int_0^\infty e^{-x}\, dx
\]
</div>

{% highlight latex %}
\sum_{i=1}^{10} t_i
\int_0^\infty e^{-x}\, dx
{% endhighlight %}

There are more operators that use @^@ and @_@ for specifying boundaries, for example @\prod@ $\prod$ and @\bigcup@ $\bigcup$. A more detailed list is shown at the end of this page.

With @\ldots@ $\ldots$ and @\cdots@ $\cdots$ one makes dots. The context determines which dots to use. For example, we write @1,2,\ldots,n@ $1,2,\ldots,n$ and @1\cdot 2 \cdots n@ $1\cdot 2\cdots n$. In general we try to choose the dots in such a way that they are at the same height as the surrounding operators. The command @\dots@ tries to accomplish this.



h2. Brackets

*Brackets* and other *delimiters* are typeset in the following way:

<div class="example">
\[
( a )\quad [ b ]\quad \{ c \}\quad \mid d \mid \quad \| e \|\quad
\langle f \rangle\quad \lfloor g \rfloor\quad
\lceil h \rceil\quad \ulcorner i \urcorner
\]
</div>

{% highlight latex %}
( a )  [ b ]  \{ c \}  | d |  \| e \| 
\langle f \rangle 
\lfloor g \rfloor 
\lceil h \rceil 
\ulcorner i \urcorner
{% endhighlight %}

You can scale these _delimiters_ automatically using @\left@, @\right@ en @\middle@.


<div class="example">
\[
\left| \sum_{n=1}^\infty a_n \right| \leq  \sum_{n=1}^\infty \left| a_n \right|
\]
</div>

{% highlight latex %}
\left| \sum_{n=1}^\infty a_n \right| \leq  \sum_{n=1}^\infty \left| a_n \right|
{% endhighlight %}

You can scale them manually using @\bigl@, @\bigr@; @\Bigl@, @\Bigr@; @\biggl@, @\biggr@; @\Biggl@, @\Biggr@. You should only do this if necessary.

\]
</div>

h2. Horizontal whitespace

Sometimes it is necessary to create some horizontal whitespace. The command @\quad@ inserts whitespace the size of the font type, so 10pt for a 10pt font. A @\qquad@ inserts twice as much whitespace. These commands also work in _text mode_. In _math mode_ we have the following specific commands:

table(table table-condensed).
|_.commando |_.lengte |
| @\,@ | 3/18 quad &emsp;&emsp;&emsp;&emsp;&emsp;&emsp;&emsp;&emsp;&emsp;&emsp;&emsp;&emsp;&emsp;&emsp;&emsp;&emsp;&emsp;&emsp;&emsp;&emsp;    |
| @\:@ | 4/18 quad |
| @\;@ | 5/18 quad |
| @\!@ | $-$3/18 quad|


<div class="example">
\[
\int y\, dx
\]
</div>

{% highlight latex %}
\int y\, dx
{% endhighlight %}

| @\mathsf@		| $\mathsf{ABCDEF}$		| $\mathsf{abcdef}$ 	| |

| @\mathbb@		| $\mathbb{ABCDEF}$		|  						| sets |

We prefer not to use these commands directly in the code, but in a new command to seperate the content and layout. For example:

<div class="example">
The complex numbers $\mathbb{C}$.
</div>

{% highlight latex %}
\documentclass[a4paper]{article}
\usepackage[dutch]{babel}
\usepackage{mathtools}
\usepackage{amssymb}

\newcommand{\field}[1]{\mathbb{#1}}
\newcommand{\CC}{\field{C}}

\begin{document}
The complex numbers $\CC$.
\end{document}
{% endhighlight %}
